# ADR 0006: Отрицательный остаток и reconciliation

- Статус: Accepted
- Дата: 2026-07-19

## Решение

Локальная операция, дающая computed quantity < 0, сначала требует человеческого подтверждения. После подтверждения событие сохраняется без clamp, projection получает `needs_reconciliation = true`, а обычный UI показывает `0` вместе с текстом «Нужно уточнить».

Исправление выполняется `INVENTORY_RECONCILED` с delta `actual_quantity - computed_quantity`. Это событие единственное очищает reconciliation flag. Старые события остаются неизменными.

## Причина

Физическое действие важнее локального предположения об остатке. Блокировка или удаление события создали бы красивое число и ложную историю — банка Шрёдингера нам не нужна.

## Последствия

Внутренний computed quantity может быть отрицательным. Любой consumer projection обязан показывать display quantity и reconciliation explanation вместе, не голое отрицательное число.
